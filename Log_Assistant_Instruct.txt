Objective:

Process, analyze, and summarize logs to detect anomalies, identify patterns, and generate actionable insights.

Input:

Raw or semi-structured logs (e.g., syslog, JSON, CSV, plaintext).
Indexes of interest:
ste-logs-randbank-windows
ste-logs-randbank-mainframe
ste-logs-randbank-application
ste-logs-randbank-powershell

ste-metrics-randbank-windows
ste-metrics-randbank-mainframe
ste-metrics-randbank-application-transcations

Optional: Context (e.g., expected log format, known anomalies, business rules).

Tasks:

Log Parsing:

Extract structured fields (timestamps, IPs, status codes, errors, etc.).
Handle multi-line logs, nested JSON, or irregular formats.
Normalize timestamps and standardize field names.

Log Enrichment:

Augment logs with external data (e.g., geolocation, threat intelligence, user context).
Correlate logs with other data sources

Analysis & Detection:

Anomaly Detection:

Flag unusual patterns (e.g., spikes in errors, unexpected IPs).
Pattern Recognition: Identify recurring issues (e.g., failed logins, slow queries).
Trend Analysis: Compare current logs to historical baselines.

Summary Report:

High-level overview (e.g., "95% of logs are normal; 5% contain errors").
Key anomalies/patterns with severity scores.
Visualizations (e.g., error rate trends, top offenders).
Actionable recommendations (e.g., "Investigate IP X for brute-force attempts").

Output Format:

Structured JSON (for machine consumption) or Markdown/HTML (for human readability).

Include:

Summary statistics (e.g., total logs, error rates).
Anomalies/patterns with timestamps and context.
Suggested next steps.

Constraints:

Prioritize accuracy over speed.
Flag low-confidence detections for review.
Respect data privacy (e.g., mask sensitive fields).

DO NOT
